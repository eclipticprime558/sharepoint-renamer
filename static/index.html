1

Register an App in Azure

One-time setup — grants FileForge permission to read (and optionally rename) your SharePoint files.

You need a Microsoft 365 admin account to complete app registration. If you're not the admin, forward these steps to your IT department.
1
Open App Registrations

Sign in to portal.azure.com, search App registrations in the top bar, then click + New registration.

2
Fill In the Registration Form

Name: something descriptive, e.g. SharePoint File Manager
Supported account types: Accounts in this organizational directory only
Leave Redirect URI blank for now. Click Register.

3
Copy Your Client ID

On the app's Overview page, copy the Application (client) ID — a GUID like xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx. You'll paste it in Step 2 below.

4
Add API Permissions

Go to API permissions → Add a permission → Microsoft Graph → Delegated permissions. Add the scopes for your use case:

Scan Only

Read-only — no changes made to SharePoint

Sites.Read.All Files.Read.All
Scan + Rename

Can read and rename files in SharePoint

Sites.ReadWrite.All Files.ReadWrite.All
5
Grant Admin Consent

Still on the API permissions page, click Grant admin consent for [your org] → Yes. Requires a Global Admin account.

6
Add a Redirect URI

Go to Authentication → Add a platform → Single-page application (SPA).
Set the Redirect URI to: loading…

FileForge never persists your access token. Auth is handled in the browser by Microsoft's MSAL library.

2

Connect to Microsoft 365

Enter your Client ID from Step 1 and sign in with your Microsoft 365 account.

From the Overview page of your Azure App Registration (Step 1, item 3).

Connected to . Continue to Step 3 to configure your scan.

6

Apply Renames

Select a folder and push approved name changes to SharePoint.

You signed in with read-only access. Sign out and sign back in with Scan & Rename to apply changes.

Only approved files with a different suggested name are renamed. SharePoint version history preserves the original name.
